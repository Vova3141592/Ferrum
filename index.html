Виды инфраструктуры
Стандартный подход сделать ИТ-инфраструктуру в компании:
- купить кучу нормальных компьютеров;
- купить кучу серверов.
Такая инфраструктура называется дивергентной: то есть раздельно всё делается. Все элементы системы отделены друг от друга. Все элементы относительно самостоятельны. Классика. Такую инфраструктуру я видел на практике в ЦБДД, в Пермской станции скорой медицинской помощи и в центре гигиены и эпидемиологии в Пермском крае.
Но можно сделать проще:
Такая инфраструктура называется конвергентной:
- компьютеры максимально простые, потому что на них ничего не делается, они нужны только для того, чтобы подключиться к виртуальным рабочим местам;
- есть один сервер для вычислений: там находится 1С без базы данных, почтовый сервер без базы данных, бездисковые виртуальные рабочие места, контроллер домена и веб-сервер без базы данных. В общем, все серверы-программы переехали на один компьютер-сервер.
- есть сервер для данных: здесь находятся всякие базы данных: для 1С, для веб-сервера, для почтового сервера и диски для виртуальных рабочих мест.
Так можно сделать с помощью двух способов: VDI и терминальный сервер.
Терминальные серверы:
- LTSP (Linux);
- просто один xrdp + многопользовательская ОС, например Xubuntu (Linux);
- RDC (Windows).
VDI:
- FreeUDS (бесплатно, наверное, Linux);
- VDI в WindowsServer;
- VMWare, там целая куча программ для того, чтобы это сделать.
